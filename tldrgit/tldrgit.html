Clone and set up remotes
  • Copy the repository's clone url from github (this should be from your fork of the repository).
  • In teminal navigate to the directory where you want to clone the repository.
  • $ git clone <url-for-clone-from-github> enter github username and password if promted.
  • $ cd <repostory-name> cd into the new directory created for the cloned repository.
  • $ git remote -v list remotes for local git repository
  • $ git remote add <remote-name> <url-for-remote> to add a remote. <remote-name> can be whatever name you want. If the new remote is a repository on github (e.g. set the repository that your fork originated from as the upstream remote), then the <url-for-remote> is that repository's clone url on github.
  • $ git remote -v should now show the added remote.

Fetch and merge changes from remote
  • $ git remote -v list remotes for local git repository
  • $ git fetch <remote-name> fetch all branches from remote-name, these branches are now accessible locally as ... <remote-name>/<branch-name> which you can merge, checkout or inspect.
  • $ git merge <remote-name>/<branch-name> merges to the active git branch
  • Push updated local branch to other remote branches if needed (e.g. use case: fetch from upstream remote, merge locally, push to origin remote)
  • $ git push <remote-name> <branch-name> where branch-name is the local branch that you want to push to the remote.

Amend a commit
  • To amend your last commit (to amend the commit message, or the files in the snapshot, or both).
  • If amending the files in the snapshot, stage the files as normal ($ git add ... or $ git rm ... to remove a tracked file).
  • $ git commit --amend this drops you into your text editor with your last commit message.
  • Edit the commit message if desired, but not required. Save and close.
  • Note that amending changes the SHA-1 of the commit. It’s like a very small rebase – don’t amend your last commit if you’ve already pushed it.

Exclude local files
  • To exclude local files in your git repository from being tracked.
  • Open the <repository>/.git/info/exclude file in your text editor.
  • Add directory and file paths to exclude. One per line.
  • E.g. add the line /directory-to-exclude to the exclude file in order to exclude the <repository>/directory-to-exclude directory.

Commit message
  • $ git commit will open your text editor where you enter a commit message, save and close - this is the preferred method as you can enter both a subject and body.
  • $ git commit -m "commit message subject" is all done in terminal but you can only enter a 1 line subject, use only if no body is needed.
  • Always separate subject from body with a blank line (so parsers know what is what).
  • E.g.
    Subject line summary of changes in imperative

    Body of commit message if needed. Give more context to the commit,
    say what changed and why. Wrap at 72 characters.
  • Subject line
    • Use imperative, e.g. add, not adds or added. A properly written subject will complete this sentance: If applied, this commit will <subject line here>. The imperative mood is written as a request or command.
    • Limit subject to 50 characters.
    • Capitalize the subject line.
    • Don't end the sunject line with a period.
  • Body
    • Use it to say what changed and why, but not how (the code does that).
    • Add any other context that explians why the commit exists or what it relates to.
    • Wrap at 72 characters so it will display nicely on an 80 column terminal with indenting.
    • - Bullets are okay too.
      - Typlically a hyphin followed by a single space.

Commit history
  • $ git log to show the commit history.
  • Press q to exit listing mode in terminal.
  • Add -p to show the difference introduced in each commit.
  • Add -2 to limit the output to the past 2 commits.
  • Add --oneline to show each commit as a single line.
  • E.g. $ git log -5 --oneline
  • Long pretty version: $ git log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative
  • You can set this up as an alias in ~/.gitconfig to run as $ git h
